Aug.9/2021
PAVAS/PASVC
When you download the project
PAVAS is set in application.yml as server.port=8080
PASVC is set in application.yml as server.port=8081

In the project configuration the contxt is not explicit set. So you set it 
as an Environment variable (Edit IntelliJ Config)  and edit environment 
variables

For PAVAS set it to
server.servlet.context-path=/dcl-apps-productavail-vas

for PASVC set it to 
server.servlet.context-path=/dcl-apps-productavail-svc

Or if you prefer set this property in the application.yml.

In PAVAS to set it to point to PASVC locally set it to

  "_comment": "Product Availability Service",
  "product.availability.service.url": "http://localhost:8081/dcl-apps-productavail-svc/",

In app.properties.json

NORMALLY it is set as follows

  "_comment": "Product Availability Service",
  "product.availability.service.url": "https://latest.dcl-caap.wdprapps.disney.com/dcl-apps-productavail-svc/",

LOGTIP
To include the version in WDPR RA logging

1. Open pom.xml file
Locate the <build> node and add the following as a child of it
<resources>
      <resource>
        <directory>src/main/resources</directory>
        <filtering>true</filtering>
      </resource>
    </resources>

2. Locate the <plugins> node and inside of it put the following
     <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-resources-plugin</artifactId>
        <version>3.2.0</version>
        <configuration>
          <delimiters>
            <delimiter>@</delimiter>
          </delimiters>
          <useDefaultDelimiters>false</useDefaultDelimiters>
        </configuration>
      </plugin>

If you see an error when building it may happen you have binary files and other
file extensions, so we need to exclude those for this reseouce mapping

Change this to look like this
     <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-resources-plugin</artifactId>
        <version>3.2.0</version>
        <configuration>
          <delimiters>
            <delimiter>@</delimiter>
          </delimiters>
          <useDefaultDelimiters>false</useDefaultDelimiters>
          <nonFilteredFileExtensions>
            <nonFilteredFileExtension>pdf</nonFilteredFileExtension>
            <nonFilteredFileExtension>xml</nonFilteredFileExtension>
            <nonFilteredFileExtension>csv</nonFilteredFileExtension>
            <nonFilteredFileExtension>jks</nonFilteredFileExtension>
            <nonFilteredFileExtension>json</nonFilteredFileExtension>
          </nonFilteredFileExtensions>
        </configuration>
      </plugin>

3. Open WdprLogAppender.properties and update Version
Version=@project.version@

NOTE: If you are creating the project using Spring Boot 2.x, this is already
configured out of the box, except you have in the resources folder some binary
files.

Install Node.js Locally with Node Version Manager (nvm)
https://heynode.com/tutorial/install-nodejs-locally-nvm

Node Version Manager - POSIX-compliant bash script to manage multiple active 
node.js versions
https://github.com/nvm-sh/nvm

This notes are for nvm V0.38.0

$ nvm list -> Reports installed versions under 'nvm' control.

$ nvm install 14 

Tells nvm to install the latest release for 14 line.

$ nvm install 14.12.0 

Installs the specific version.

$ nvm use 14 -> Switch to this installed version. If there are two or more it
gets the latest version in that branch, e.g., if we have 14.15.4 and 14.17.0 
installed, it will favor 14.17.0 to use. 

$ nvm use system -> Indicates to work with the node version installed via the 
nodejs setup wizard.
